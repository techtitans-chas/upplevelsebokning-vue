## Key Design Decisions

1. **Composable + Component Split**
   - Composable: Pure state management
   - Component: Pure presentation
   - Separation of concerns

2. **Teleport to Body**
   - Ensures proper stacking context
   - Avoids z-index conflicts
   - Global positioning freedom

3. **Separate Animation States**
   - Entry: Fast (300ms, ease-out)
   - Exit: Slow (300ms, ease-in)
   - Better perceived flow

4. **Auto-ID Generation**
   - Simple counter-based
   - No UUID needed for this use case
   - Lightweight

5. **Variant System**
   - 5 pre-defined variants
   - Semantic (success/error/warning/info)
   - Easy to extend

6. **Flexible Positioning**
   - 2 positions Ã— 3 alignments = 6 combos
   - Covers all common use cases
   - Simple, predictable
